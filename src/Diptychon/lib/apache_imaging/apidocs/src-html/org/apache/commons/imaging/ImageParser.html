054    /**
055     * Provides the abstract base class for all image reading and writing
056     * utilities.  ImageParser implementations are expected to extend this
057     * class providing logic for identifying and processing data in their
058     * own specific format.   Specific implementations are found
059     * under the com.apache.commons.imaging.formats package.
060     * <h3>Application Notes</h3>
061     * <h4>Format support</h4>
062     * For the most recent information on format support for the
063     * Apache Commons Imaging package, refer to
064     * <a href="http://commons.apache.org/imaging/formatsupport.html">Format Support</a>
065     * at the main project development web site.
066     * <h4>On the accuracy of this Javadoc</h4>
067     * The original authors of this class did not supply documentation.
068     * The Javadoc for this class is based on inspection of the 
069     * source code.  In some cases, the purpose and usage for particular
070     * methods was deduced from the source and may not perfectly reflect
071     * the intentions of the original. Therefore, you should not assume 
072     * that the documentation is perfect, especially in the more obscure
073     * and specialized areas of implementation.
074     * <h4>The "Map params" argument</h4>
075     * Many of the methods specified by this class accept an argument of
076     * type Map giving a list of parameters to be used when processing an
077     * image. For example, some of the output formats permit the specification
078     * of different kinds of image compression or color models. Some of the
079     * reading methods permit the calling application to require strict
080     * format compliance.   In many cases, however, an application will not
081     * require the use of this argument.  While some of the ImageParser 
082     * implementations check for (and ignore) null arguments for this parameter,
083     * not all of them do (at least not at the time these notes were written).
084     * Therefore, a prudent programmer will always supply an valid, though
085     * empty instance of a Map implementation when calling such methods.
086     * Generally, the java HashMap class is useful for this purpose.
087     * <p>Additionally, developers creating or enhancing classes derived
088     * from ImageParser are encouraged to include such checks in their code.
089     */

558         * Writes the content of a BufferedImage to the specified output
559         * stream.
560         * <p>The params argument provides a mechanism for individual
561         * implementations to pass optional information into the parser.
562         * Not all formats will support this capability.  Currently,
563         * some of the parsers do not check for null arguments. So in cases
564         * where no optional specifications are supported, application
565         * code should pass in an empty instance of an implementation of 
566         * the map interface (i.e. an empty HashMap).
567         * @param src An image giving the source content for output
568         * @param os A valid output stream for storing the formatted image
569         * @param params A non-null Map implementation supplying optional, 
570         * format-specific instructions for output 
571         * (such as selections for data compression, color models, etc.)
572         * @throws ImageWriteException In the event that the output format
573         * cannot handle the input image or invalid params are specified.
574         * @throws IOException In the event of an write error from
575         * the output stream.
576         */
577        public void writeImage(final BufferedImage src, final OutputStream os, final Map<String,Object> params)
578                throws ImageWriteException, IOException {
579            try {
580                os.close(); // we are obligated to close stream.
581            } catch (final Exception e) {
582                Debug.debug(e);
583            }
584    
585            throw new ImageWriteException("This image format (" + getName()
586                    + ") cannot be written.");
587        }

917         * Indicates whether the ImageParser implementation can accept

919         * @param type An instance of ImageFormat.

923            final ImageFormat types[] = getAcceptedTypes();
924

934         * Indicates whether the ImageParser implementation can accept

936         * @param file An valid file reference.

940            return canAcceptExtension(file.getName());

943        /**
944         * Indicates whether the ImageParser implementation can accept
945         * the specified file name based on its extension.
946         * @param filename An valid string giving a file name or file path. 
947         * @return If the parser can accept the format, true; otherwise, false.
948         */
949        protected final boolean canAcceptExtension(final String filename) {
950            final String exts[] = getAcceptedExtensions();
951            if (exts == null) {
952                return true;
953            }
954    
955            final int index = filename.lastIndexOf('.');
956            if (index >= 0) {
957                String ext = filename.substring(index);
958                ext = ext.toLowerCase();
959    
960                for (final String ext2 : exts) {
961                    if (ext2.toLowerCase().equals(ext)) {
962                        return true;
963                    }
964                }
965            }
966            return false;
967        }
